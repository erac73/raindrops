Que es Rain Drops?

Rain Drops es un modelo de almacenamiento distribuido basado en criptografia de umbral (Shamir's Secret Sharing). Los datos se fragmentan en micro-unidades criptograficas llamadas gotas (drops) que individualmente no tienen significado. Los datos originales solo pueden reconstruirse cuando un numero suficiente de gotas convergen bajo condiciones verificadas.

Esto no es cifrado encima de almacenamiento. La confidencialidad es una propiedad estructural del modelo.

[K]

Secreto Perfecto

Con menos de K gotas, el secreto esta oculto informaticamente. Ni computadoras cuanticas pueden romperlo.

[N]

Umbral K-de-N

Los datos se dividen en N gotas. Cualquier K gotas reconstruyen; K-1 no revelan nada. Resiliente a fallos de nodos.

[TTL]

Expiracion Temporal

Cada gota tiene un TTL. Las gotas expiradas se eliminan automaticamente. Control de acceso temporal por diseno.

[HMAC]

Integridad Verificada

Cada gota lleva un HMAC-SHA256. El Nodo Testigo verifica todas las gotas antes de reconstruir.

[ARM]

Docker Multi-Arch

Imagenes ARM64 y AMD64. Despliegue en Raspberry Pi, servidores x86 o cloud.

[MET]

Metricas Prometheus

Todos los nodos exponen metricas, health checks, docs Swagger y dashboard web.

Componentes

  • raindrops-core — Libreria criptografica (SSS, AES-GCM, HMAC, RainMap)
  • raindrops-storage — Nodo de almacenamiento Spring Boot con replicacion y reaper TTL
  • raindrops-witness — Nodo testigo para verificacion segura de reconstruccion
  • RainClient SDK — Cliente Java de alto nivel para operaciones DROP/RECONSTRUCT

Demo Rapido

Prueba Rain Drops en 60 segundos

Paso 1 — Abrir la Demo

Haz clic en el boton para abrir la demo interactiva:

Abrir Demo →

Paso 2 — Configurar la URL del Witness Node

En la parte superior de la demo, configura la URL del Witness Node:

http://localhost:9080        # local
http://<ip-pi>:9080          # Raspberry Pi

El Witness Node coordina el almacenamiento y la reconstruccion. Debe estar ejecutandose.

Paso 3 — Almacenar Datos

  1. Escribe cualquier texto en el campo Data to store (ej: Mi contrasena secreta)
  2. Configura los parametros:
    • N (total drops) — Total de fragmentos a crear (recomendado: 5)
    • K (threshold) — Minimo de gotas necesarias para reconstruir (recomendado: 3)
    • TTL (days) — Los drops expiran despues de este tiempo (recomendado: 30)
  3. Haz clic en Store
  4. La demo retorna:
    • RainMap ID — Indice cifrado de donde estan tus drops
    • Master Key — Tu clave de descifrado (guardala!)

Estos dos valores son todo lo que necesitas para reconstruir despues. El Witness Node no almacena nada.

Paso 4 — Reconstruir

  1. Los campos RainMap ID y Master Key se autocompletan despues de almacenar
  2. Haz clic en Reconstruct
  3. Los datos originales aparecen en el panel de respuesta

Paso 5 — Verificar que Funciona

Prueba estos experimentos:

  • Elimina un drop: Borra un nodo de almacenamiento y reconstruye — sigue funcionando (K de N)
  • Clave incorrecta: Cambia un caracter en la Master Key — la reconstruccion falla
  • Drop expirado: Pon TTL=1, espera un dia, reconstruye — los drops desaparecieron

Que Pasa por Detras

STORE:
  Cliente → Witness → [AES-256-GCM cifrar] → [Shamir SSS dividir] → Distribuir a N nodos
  Witness retorna: rainMapId + masterKey

RECONSTRUCT:
  Cliente → Witness (rainMapId + masterKey) → [Localizar drops] → [Verificar HMAC] → [Interpolacion de Lagrange] → [AES descifrar]
  Witness retorna: datos originales

Despliegue

Tres nodos de almacenamiento (puertos 9081-9083) + un nodo testigo (puerto 9080) orquestados via Docker Compose. Soporte nativo ARM64 para Raspberry Pi.
