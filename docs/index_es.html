Que es Rain Drops?

Rain Drops es un modelo de almacenamiento distribuido basado en criptografia de umbral (Shamir's Secret Sharing). Los datos se fragmentan en micro-unidades criptograficas llamadas gotas (drops) que individualmente no tienen significado. Los datos originales solo pueden reconstruirse cuando un numero suficiente de gotas convergen bajo condiciones verificadas.

Esto no es cifrado encima de almacenamiento. La confidencialidad es una propiedad estructural del modelo.

[K]

Secreto Perfecto

Con menos de K gotas, el secreto esta oculto informaticamente. Ni computadoras cuanticas pueden romperlo.

[N]

Umbral K-de-N

Los datos se dividen en N gotas. Cualquier K gotas reconstruyen; K-1 no revelan nada. Resiliente a fallos de nodos.

[TTL]

Expiracion Temporal

Cada gota tiene un TTL. Las gotas expiradas se eliminan automaticamente. Control de acceso temporal por diseno.

[HMAC]

Integridad Verificada

Cada gota lleva un HMAC-SHA256. El Nodo Testigo verifica todas las gotas antes de reconstruir.

[ARM]

Docker Multi-Arch

Imagenes ARM64 y AMD64. Despliegue en Raspberry Pi, servidores x86 o cloud.

[MET]

Metricas Prometheus

Todos los nodos exponen metricas, health checks, docs Swagger y dashboard web.

[RL]

Audit Logging

[HP]

Health Checks

Indicadores detallados: memoria, disco, uptime, conteo de nodos. Listo para Prometheus.

Componentes

  • raindrops-core — Libreria criptografica (SSS, AES-GCM, HMAC, RainMap)
  • raindrops-storage — Nodo de almacenamiento Spring Boot con replicacion y reaper TTL
  • raindrops-witness — Nodo testigo para verificacion segura de reconstruccion
  • RainClient SDK — Cliente Java de alto nivel para operaciones DROP/RECONSTRUCT

Inicio Rapido

git clone https://github.com/erac73/raindrops.git
cd raindrops
mvn -f raindrops-fase1/raindrops/pom.xml install
docker compose -f docker-compose.yml up -d --build

Despliegue

Tres nodos de almacenamiento (puertos 9081-9083) + un nodo testigo (puerto 9080) orquestados via Docker Compose. Soporte nativo ARM64 para Raspberry Pi.
